# SC Like

**SC Like** — консольная утилита для пакетной обработки списка треков из `tracks.txt`:  
она **ищет** каждый трек в SoundCloud и затем **ставит лайки** найденным трекам, сохраняя прогресс и кэш, чтобы можно было безопасно **возобновлять работу** после остановки.

---

## Зачем это нужно

- Быстро лайкнуть большую подборку треков из текстового списка (например, перенос аудитеки из других сервисов)
- Не терять прогресс при ошибках/закрытии программы.
- Не перегружать API повторными поисками: найденные треки кэшируются.

---

## Что делает программа

1. **Авторизация SoundCloud по OAuth 2.1 (Authorization Code + PKCE)** через ваш SoundCloud App.
2. Читает список из `tracks.txt` (формат строк ниже).
3. **Ищет** каждый трек в SoundCloud и сохраняет найденные в локальный кэш.
4. **Ставит лайки** найденным трекам.
5. При повторном запуске:
    - может использовать кэш найденных,
    - может продолжить обработку с места остановки,
    - может начать обработку заново с нуля (по выбору).

---

## Важные ограничения SoundCloud (это нормально)

SoundCloud использует антиспам/антифрод ограничения:

- **HTTP 429 Too Many Requests** — API ограничивает скорость действий (лайки особенно).  
  Документация: https://developers.soundcloud.com/docs/api/rate-limits.html
- У лайков есть **суточные лимиты** и возможные временные блокировки при подозрительной активности.  
  Help Center: https://help.soundcloud.com/hc/en-us/articles/115003568968-Liking-tracks-or-playlists
- Если вы видите “suspicious activity”, лайки/подписки могут **откатываться** — это ограничения аккаунта, не баг программы.

**Поведение SC Like:**
- если при лайке прилетает **429**, программа **сразу останавливается**, сохраняя прогресс (чтобы вы продолжили позже).

## Формат `tracks.txt`

Файл должен лежать рядом с программой: `./tracks.txt`  
Каждая строка:

```
Исполнитель - Название
```

Пример:

```
Linkin Park - Somewhere I Belong
Daft Punk - Harder Better Faster Stronger
```

Можно добавлять комментарии, строки начинающиеся с `#` будут игнорироваться.

## Файлы и кэш (data/*)

Программа хранит всё в папке `data/`:

- `data/configs/*.json` — профили конфигураций (client_id / client_secret / redirect_uri)
- `data/tokens.json` — OAuth токены (access/refresh)
- `data/found_tracks.json` — кэш найденных треков (для лайкинга)
- `data/state_search.json` — прогресс поиска (с какой строки продолжать)
- `data/state_likes.json` — прогресс лайков (какие треки уже обработаны)
- `data/not_found.txt` — строки из `tracks.txt`, которые не удалось сопоставить
- `data/errors.log` — ошибки (ответы API и сообщения)

## Принципы работы (коротко)

### Поиск
- Идёт по `tracks.txt`, пытается найти лучшее совпадение (эвристический матчинг по названию и исполнителю).
- Сохраняет найденные треки **инкрементально**, чтобы при остановке продолжить.
- Может продолжить с места остановки по `state_search.json`.

### Лайки
- Для каждого найденного трека:
    - проверяет локальный `state_likes.json` (уже обработан — пропускает),
    - делает best-effort проверку “уже лайкнут?”,
    - ставит лайк,
    - сохраняет результат в `state_likes.json`.
- При **429** — немедленный стоп с сохранением результатов.

## Как создать приложение SoundCloud (получить Client ID / Secret)

SoundCloud требует, чтобы действия от имени пользователя (лайки) делались через **OAuth 2.1** и зарегистрированное приложение.

Официальная документация по OAuth:  
https://developers.soundcloud.com/docs  
и блог о миграции на OAuth 2.1:  
https://developers.soundcloud.com/blog/oauth-migration/

### Шаги
1. Откройте портал разработчика SoundCloud (https://soundcloud.com/you/apps) и зарегистрируйте приложение (Create App).
2. Получите:
    - `client_id`
    - `client_secret`
3. Укажите **Redirect URI** (важно: должен совпадать 1:1 с тем, что введёте в программе).  
   Рекомендуемый локальный вариант:
   ```
   http://127.0.0.1:53682/callback
   ```
4. Сохраните изменения.

## Частые проблемы

### “Получен 429 при лайке”
Это ограничение SoundCloud. Решения:
- подождать и продолжить позже (SC Like сохранит прогресс)
- уменьшить активность (лайки вручную/скриптом) в течение суток
- убедиться, что нет VPN/прокси/скачков IP (это часто триггерит антифрод)

### “Лайк пропадает / suspicious activity”
Это антифрод SoundCloud на аккаунте. Программа не может это обойти.  
Обычно помогает пауза 24–72 часа, смена пароля/выход из всех сессий и стабилизация входов.
